# Project Board Setup

Live project:

- JonBranding Web OS (GitHub user project #2, private)

Recommended GitHub Project columns:

- Inbox
- Backlog
- Ready
- In progress
- Review
- Ready to deploy
- Done

Recommended custom fields:

- Priority: `P0`, `P1`, `P2`, `P3`
- Area: `UX`, `UI`, `CRO`, `SEO`, `Security`, `Performance`, `Content`, `CI/CD`
- Surface: `Homepage`, `Service page`, `Blog`, `Contact modal`, `API`, `Deployment`
- Status: GitHub project status

Automation rules to enable in GitHub Projects:

- New issue -> Inbox
- PR opened -> In progress
- PR ready for review -> Review
- PR merged -> Done

Useful saved views:

- P0/P1 only
- Conversion roadmap
- Design system work
- Security and dependency work
- Current sprint
